# Code Style Rules

- Use semantic HTML5 elements (header, main, section, footer, nav)
- Tailwind classes: use responsive prefixes (sm:, md:, lg:) for mobile-first design
- React state: prefer useState and useEffect hooks, no class components
- Variable and function names in English (camelCase)
- UI text and content in Hebrew
- Indent with 4 spaces
- Always add alt text to images for accessibility
- Use aria-labels on interactive elements for screen readers
